LastError Exception
The last error "swallowed" by the cache. All KVLite caches, by definition, try to swallow as much exceptions as possible, because a failure in the cache should never harm the main application. This is an important rule. This property might be used to expose the last error occurred while processing cache items. If no error has occurred, this property will simply be null. Every error is carefully logged using the provided PommaLabs.KVLite.ICache.Log, so no information is lost when the cache swallows the exception.

The maximum number of parent keys each item can have. SQL based caches support up to three parent key per item. There is no fast way to track many keys without adding lots of indexes, which would make all write operations slower; therefore, the number of parent keys has to be limited to three.
